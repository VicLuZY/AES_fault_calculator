/goal Build FaultCalc into a professional offline Rust + WebAssembly short-circuit workstation.

Context
You are continuing from the seed package `ieee_faultcalc_rust_wasm`. The user wants a serious, offline, portable fault-current calculator with a professional custom GUI that renders and modifies the system setup. The previous Go WASM prototype was intentionally replaced. The long-term direction is Rust for the engineering calculation core, native CLI, browser WASM, and durable testability.

Primary outcome
Deliver a production-quality offline workstation that can:
1. Model an electrical distribution system programmatically and graphically.
2. Calculate available fault current at every bus.
3. Support 3PH, SLG, LL, and DLG faults using sequence networks.
4. Run as a native CLI and as a browser-based Rust WASM app.
5. Open without internet access.
6. Export auditable JSON and CSV reports.
7. Keep the engineering method explicit, testable, and extendable.

Current package structure
- `crates/faultcalc-core`: case model, validation, solver, report export.
- `crates/faultcalc-cli`: native CLI, local file server, standalone HTML embedding tool.
- `crates/faultcalc-wasm`: direct Rust WASM ABI without wasm-bindgen.
- `web/index.template.html`: professional custom GUI template.
- `web/index.html`: development GUI that fetches `faultcalc.wasm` when served.
- `cases/sample.json`: sample electrical system.
- `docs/calculation-method.md`: current method notes.

First build commands
Run these first and fix any compile or lint issues before adding features:

```bash
rustup target add wasm32-unknown-unknown
cargo test --workspace
cargo build -p faultcalc-cli --release
mkdir -p bin
cp target/release/faultcalc bin/faultcalc
cargo build -p faultcalc-wasm --release --target wasm32-unknown-unknown
cp target/wasm32-unknown-unknown/release/faultcalc_wasm.wasm web/faultcalc.wasm
bin/faultcalc embed-wasm web/index.template.html web/faultcalc.wasm web/faultcalc_workstation.html
bin/faultcalc calc cases/sample.json --json out/report.json --csv out/report.csv
```

Do not introduce Python as a runtime dependency. Python may be used only for one-off local developer scripting if absolutely necessary, but the final product must be Rust plus browser assets.

Design principles
- Keep the calculation engine independent from the GUI.
- Make unit conversions explicit and test-covered.
- Never bury engineering assumptions inside the UI.
- Treat every calculation result as auditable data.
- Prefer deterministic, typed structures over ad hoc maps.
- Keep the app offline-first and portable.
- Do not require a cloud service, database, or telemetry.
- Do not embed copyrighted IEEE, ANSI, NEMA, or manufacturer tables unless the user supplies licensed data.
- When exact standard tables are needed, create a data-loader interface and document the required source, version, and licence boundary.

Validation and testing
Do not treat visual success as calculation correctness. Build tests from the inside out.

Required tests:
- Complex arithmetic unit tests.
- Per-unit base conversion tests.
- Matrix solver tests.
- Sequence network driving-point impedance tests.
- Fault formula tests for 3PH, SLG, LL, DLG.
- Sample case snapshot tests.
- CLI vs WASM parity tests.
- JSON import/export round-trip tests.
- Disabled branch/source tests.
- Zero-sequence path absent tests.
- Fault impedance non-zero tests.

Use tolerances explicitly. Document expected values and how they were derived.

WASM ABI contract
The seed uses a direct ABI instead of wasm-bindgen:
- `faultcalc_alloc(len) -> ptr`
- `faultcalc_free(ptr, len)`
- `faultcalc_calculate(ptr, len) -> i32`
- `faultcalc_sample() -> i32`
- `faultcalc_version() -> i32`
- `faultcalc_output_ptr() -> ptr`
- `faultcalc_output_len() -> len`

Keep this ABI stable unless there is a compelling reason to adopt wasm-bindgen. The direct ABI keeps the offline bundle simple and avoids JavaScript glue dependencies.
